One Platform Foundation

We want to make it easier for Android developers to submit their applications to several application stores. Our first project is AppDF (Application Description File) - a format for describing Android application in one file.

AppDF

Uploading Android application to several appstores could be time consuming operation. AppDF format is designed to simplify this process. You describe your Android application once, create a simple AppDF archive that includes the XML description, APK file(s), screenshots, app icon, promo images, etc and just upload this AppDF file to all the stores that support AppDF uploading.

What Is AppDF File?

AppDF file is a ZIP archive with .appdf extension. This archive must contain description.xml file with application description (title, description text, screenshots, etc.). This description.xml file contains links to the other files in the archive like APK file(s), screenshots, promo graphics, video, etc. The main sections of the description.xml file are:

  • Category information
  • Text description (title, short description, full description, keywords, etc)
  • Images (app icon, screenshots, promo images)
  • Videos
  • Contant rating information
  • Availability (country list, etc)
  • Prices
  • Requirements (you can specify supported devices if different from the APK file manifest)
  • Testing instructions
  • Consent
  • Customer-support information
  • Store specific section

Specification

You can find current specification draft at GitHub. There is also a formal XMS Scheme (XSD) description of the used XML format. Current version is 0.92 and we plan to freeze the specification before the end of February.

Frequently Asked Questions

But will the Android applications stores support AppDF?

We have just started the project and already have commitment from three big appstores that they will support AppDF.

I represent an Android application store and we want to support AppDF, what should we do?

You can just take current specification draft and implement it in your store. But we suggest joining our mailing list and contacting us. We are developing reference AppDF parsers now that could make it easier for you to integrate AppDF support to your store.

What is source code licese?

All source code is available under Apache 2 license.

How can I create an AppDF file for my Android app?

Right now you need to do it manually. We are working on a webpage where you can submit information about your app and it will generate an AppDF file for you.
